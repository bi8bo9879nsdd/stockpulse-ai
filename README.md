# StockPulse AI

## 部署到 Streamlit Community Cloud

项目已经整理为可直接部署到 Streamlit Community Cloud 的结构。上传 GitHub 后：

1. 打开 [Streamlit Community Cloud](https://share.streamlit.io/) 并连接 GitHub。
2. 点击 `Create app`，选择仓库和分支。
3. 将入口文件设置为 `app.py`。
4. 如需优先使用 Finnhub，在 `Advanced settings` 的 `Secrets` 中填写：

```toml
FINNHUB_API_KEY = "你的 Finnhub API Key"
```

5. 点击 `Deploy`。部署完成后会获得 `https://你的子域名.streamlit.app` 公网链接，可直接在手机浏览器中访问。

仓库根目录中的 `requirements.txt` 会用于安装 Python 依赖，`.streamlit/config.toml` 会用于云端运行配置。请勿把 `.streamlit/secrets.toml` 上传到 GitHub。

### 云端数据说明

`data/*.json` 已加入 `.gitignore`，真实持仓、提醒规则和模拟交易记录不会上传到 GitHub。云端运行时仍可通过网页录入数据，但这些 JSON 文件保存在当前 Streamlit 运行实例中，重新部署、重启或实例回收后可能重置。公开链接中的运行时数据也可能被其他访问者看到或修改，因此不要在公开应用中录入敏感持仓。后续如需可靠的私有云端保存，应接入带用户鉴权的数据库。

美股实时投研与风险提醒助手。项目使用 Streamlit 构建界面，通过多数据源架构获取美股报价与历史价格数据，适合作为个人投研看板的 MVP。

## 功能

- 输入并保存自选股列表，例如 `NVDA`、`MU`、`MRVL`、`QQQ`、`AVGO`
- 展示当前价格、日涨跌幅、成交量、MA5、MA20、RSI（14）
- 展示每只股票当前使用的数据源与数据状态
- 使用规则模板生成中文 AI 投研简报
- 批量导入价格提醒规则，并结合当前行情判断是否触发
- 将自选股、持仓和提醒规则保存到本地 JSON 文件
- 支持按 `30 秒`、`1 分钟`、`5 分钟` 自动刷新行情
- 根据本地持仓配置自动生成止盈止损计划和风险提醒
- 基于持仓、提醒规则和行情生成只读交易计划草稿，并支持导出 Markdown 和 JSON
- 使用本地 Paper Trading 账本练习限价单，不连接真实券商账户
- 在网页侧边栏批量导入、清空、新增、更新和删除真实持仓

## 数据源

应用通过 `data_provider.py` 统一管理行情数据：

1. 报价优先从 Finnhub 获取。
2. 如果没有配置 `FINNHUB_API_KEY`，或 Finnhub 请求失败，自动回退到 yfinance。
3. 技术指标所需的历史日线数据暂时由 yfinance 提供。
4. 如果当前价格为空、为 `0`，或与前收盘价偏差超过 `30%`，数据状态会标记为“异常”。

yfinance 仅作为备用报价数据源和 MVP 阶段的历史数据源。它可能存在延迟，不适合直接用于高频交易或严格实时监控。

## 安装

建议使用 Python 3.10 或更高版本。

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install -r requirements.txt
```

## 配置 Finnhub API Key

从 Finnhub 获取 API Key 后，在运行应用前设置环境变量：

```powershell
$env:FINNHUB_API_KEY="你的 Finnhub API Key"
```

环境变量只对当前 PowerShell 会话生效。未设置时，应用仍然可以使用 yfinance 运行。

## 运行

### Windows 一键启动

完成首次安装后，直接双击项目目录中的 `start_stockpulse.bat`。脚本会自动使用项目本地虚拟环境启动 Streamlit，并打开本地网页。

```text
start_stockpulse.bat
```

默认地址为 `http://localhost:8501`。关闭启动脚本打开的命令窗口，即可停止本地服务。

### PowerShell 启动

需要手动启动时，也可以运行：

```powershell
streamlit run app.py
```

浏览器会自动打开应用。默认地址通常为 `http://localhost:8501`。

侧边栏提供“自动刷新行情”开关。开启后，可以选择 `30 秒`、`1 分钟` 或 `5 分钟` 的刷新间隔。页面会显示最近一次行情刷新时间。自动刷新不会清空尚未提交的 JSON 输入内容。

## 本地数据

首次运行后，应用会自动创建本地 JSON 配置，不需要数据库：

- `data/watchlist.json`：保存自选股列表
- `data/portfolio.json`：持久化保存网页录入的真实持仓，字段包括 `ticker`、`shares`、`cost_basis`、`risk_level`、`strategy_type`、`notes`
- `data/alerts.json`：持久化保存网页导入的价格提醒规则
- `data/paper_account.json`：保存模拟账户资金与回撤状态
- `data/paper_orders.json`：保存所有本地模拟订单
- `data/paper_positions.json`：保存本地模拟持仓

运行应用后，在左侧栏的“批量导入真实持仓 JSON”区域粘贴完整 JSON 数组，然后点击“导入并保存持仓”。系统会校验并覆盖保存全部持仓；如果同一个股票代码出现多次，以后面的数据为准。也可以点击“清空当前持仓”将持仓重置为空数组。

```json
[
  {
    "ticker": "NVDA",
    "shares": 10,
    "cost_basis": 100.5,
    "strategy_type": "成长股",
    "risk_level": "中",
    "notes": "手动录入备注"
  }
]
```

需要单独调整一只持仓时，可以展开侧边栏中的“单只持仓添加/更新”面板。如果股票代码已经存在，应用会更新该持仓；也可以在折叠面板中选择股票代码并删除持仓。

网页录入的数据会保存在 `data/portfolio.json` 中。应用启动时会读取该文件，并根据真实持仓生成“我的真实持仓表”和止盈止损计划。应用不会自动连接券商账户，也不会执行任何交易。

`strategy_type` 支持“核心持仓”“核心波段”“高波动题材”“高风险小票”“短线交易”“周期股”“成长股”“杠杆/高波动产品”“中低波动持仓”“反转股”“自定义策略”。JSON 导入也支持填写其他自定义策略类型，系统会自动标准化为“自定义策略”，并在导入结果中提示对应股票代码。

`risk_level` 支持“低”“中”“中高”“高”“极高”。JSON 导入中的未知风险等级会自动标准化为“中”。应用会根据策略类型、风险等级、成本价和 MA20 自动生成止损线，并按成本价上方 `10%`、`20%`、`30%` 生成三级止盈线。

## 批量导入价格提醒

在左侧栏的“批量导入价格提醒 JSON”区域粘贴完整 JSON 数组，然后点击“导入并保存提醒”。系统会校验并覆盖保存全部提醒规则；如果 `ticker + alert_type + direction` 重复，以后面的数据为准。点击“清空当前提醒”可以将提醒重置为空数组。

```json
[
  {
    "ticker": "NVDA",
    "alert_type": "跌破止损线",
    "target_price": 100,
    "direction": "below",
    "message": "请重新评估持仓风险",
    "enabled": true
  }
]
```

`alert_type` 支持“跌破止损线”“接近止损线”“达到第一止盈”“达到第二止盈”“保护利润”“高风险观察”。`direction` 支持 `below` 和 `above`。`enabled` 缺失时默认为 `true`。需要单独调整一条提醒时，可以展开侧边栏中的“单条价格提醒添加”面板。

## 说明

- 当前提醒在页面加载或手动刷新时检查，不会在应用关闭后后台推送。
- AI 投研简报目前由规则模板生成，不构成投资建议。
- 本项目仅用于投研信息整理，不构成投资建议。

## 交易计划草稿

页面会基于 `portfolio.json`、`alerts.json`、当前行情和止盈止损计划生成交易计划草稿。每条草稿包含股票代码、动作、建议数量、订单类型、草稿限价、原因和风险等级。

草稿动作仅允许 `WATCH`、`REDUCE`、`TAKE_PROFIT`、`STOP_LOSS`。系统不生成 `BUY` 或 `SELL` 指令，不连接任何券商账户，也不会自动下单。所有交易计划必须由用户手动确认后，自行在券商 App 中执行。

页面提供 Markdown 和 JSON 两种导出按钮，便于用户自行留档和复核。

## 风险免责声明

本项目仅用于投研信息整理、风险提示和交易计划草稿记录，不构成投资建议、交易建议或收益承诺。行情数据可能延迟或异常，自动生成的止盈止损线和交易计划草稿可能不适合用户的实际情况。用户必须独立判断风险，并自行承担所有投资决策与交易结果。

## Paper Trading

页面中的“模拟交易 / Paper Trading”模块是完全本地的练习账本，不连接真实券商账户，不产生真实订单，不构成投资建议。

模拟账户初始资金默认为 `100000` 美元，可以在网页中修改。点击“初始化模拟账户”会重置模拟账户，并清空模拟订单与模拟持仓。

手动模拟订单只支持 `BUY`、`SELL` 和 `LIMIT` 限价单，不支持市价单。系统使用当前行情判断限价单是否可以模拟成交：买入限价大于等于当前价时可成交，卖出限价小于等于当前价时可成交。

点击“从真实持仓初始化模拟持仓”后，系统会根据 `portfolio.json` 创建本地模拟持仓，并从初始资金中扣除持仓成本。如果真实持仓成本总额超过初始模拟资金，系统会提示错误。

点击“自动执行模拟交易”后，系统会结合当前行情、`paper_positions.json`、`alerts.json` 和止盈止损计划执行本地模拟交易：

- 触发止损线时模拟卖出 `50%` 持仓
- 达到第一止盈时模拟卖出 `30%` 持仓
- 达到第二止盈时模拟卖出 `30%` 持仓
- 高风险观察仅记录 `WATCH` 信号
- 持仓不足 `1` 股时记录跳过原因

所有模拟订单都会记录订单 ID、时间、股票代码、动作、数量、订单类型、限价、成交价、状态、原因和来源。

页面会展示模拟账户总资产、现金、持仓市值、已实现盈亏、未实现盈亏、累计收益率、当前模拟持仓和历史模拟订单。模块还会生成中文模拟交易复盘报告，并支持导出订单 JSON 和 Markdown 报告。

侧边栏提供“开启自动模拟交易”开关，以及 `30 秒`、`1 分钟`、`5 分钟` 三档检查间隔。开启后，页面会按设定间隔自动刷新行情、检查 `alerts.json` 并执行本地模拟交易。页面会显示最近一次自动检查时间。

为避免重复交易，同一个 `ticker + alert_type` 在同一交易日只会自动执行一次。执行记录会写入 `data/paper_orders.json`。关闭网页后，自动轮询不会在后台继续运行。
